### 🧠 Who I am

> *"The best way to understand a system is to break it, then fix it."*

I'm a **Field IT Support Engineer** based in Sri Lanka 🇱🇰 — someone who refuses to be boxed in by a job title. By day, I keep systems running, solve problems fast, and support users under pressure. By night (and weekends), I'm deep in my homelab — building, breaking, and rebuilding infrastructure just to truly understand how it works.

I don't just consume tech — I **build** with it. Here's what drives me:

- 🔧 **Hands-on over theory** — If I can build it and break it myself, I understand it 10× better
- 🌐 **Networks fascinate me** — From VLANs to routing protocols, the logic of how packets travel is endlessly interesting
- 🏗️ **Self-hosted everything** — 20+ Docker containers running on a homelab built entirely from second-hand hardware
- 🤝 **Open source learner** — Everything I learn, I document publicly — so others can learn too
- 🤖 **AI-augmented workflow** — I use Claude Code and GitHub Copilot daily to move faster and learn smarter

**The mindset in action:** I don't own my internet connection — so I engineered a wireless uplink bridge and routed full LAN access into my homelab. Problem? Solved. That's how I think.

### 🖥️ Homelab - Erenyx Lab

A personal infrastructure environment built entirely from second-hand hardware —
designed for hands-on learning, real-world problem solving, and running self-hosted services.

| Device                  | Role                        | Key Stack                           |
| ----------------------- | --------------------------- | ----------------------------------- |
| 🖥️ HP Z440 Workstation  | Proxmox hypervisor          | EVE-NG CE, TrueNAS CE               |
| 🟦 Intel NUC8 i7 (NUC1) | Primary Docker server       | 20+ containerised services          |
| 🟩 Intel NUC8 i3 (NUC2) | Testing and experiments     | Disposable environments             |
| 🍓 Raspberry Pi 4B      | Edge node                   | Lightweight always-on services      |
| 📡 TP-Link AX23         | Access point and LAN switch | Erenyx WiFi and LAN client access   |

> 📖 Full documentation, network topology, and problem/solution write-ups →
